# Vercel Frontend IA Shell Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the first Vercel-ready frontend shell for the accounting product, focused on information architecture, daily workflows, Bill Center, cash flow, budget review, and future Supabase integration.

**Architecture:** Create a new `apps/web` Next.js App Router application without replacing the current Google Apps Script MVP. Use typed mock data that mirrors the Supabase schema contract so the UI can be verified before live database wiring.

**Tech Stack:** Next.js App Router, React, TypeScript, CSS Modules or global CSS, Node.js test runner or Vitest for pure utility tests, Playwright/browser verification after the dev server starts.

---

## Context

The current repository does not contain a Vercel or Next.js app. The existing production MVP remains Google Apps Script plus Google Sheet. This plan creates a new frontend workspace under `apps/web` and does not deploy Vercel, push GitHub, or change Apps Script runtime behavior.

The product direction is:

- Single-user private first.
- Keep household/team fields visible in the data model for later sharing.
- Google Sheet remains active during dual-track transition.
- Supabase becomes the next data foundation.
- Bill Center is the user-facing credit-card area.
- Payment schedules remain low-level source data.
- `credit_card_bill_estimates` is a physical Supabase table in the first version, regenerated by the app or import process, while staying compatible with a future view/materialized-view approach.

## Design Direction

Use a quiet operational finance interface. The first screen is the actual working dashboard, not a landing page. Avoid marketing hero sections, decorative gradients, nested cards, and playful visuals. The app should feel like a dense but calm private finance console:

- Left navigation for stable product areas.
- Top month selector and household scope indicator.
- First viewport shows this month cash position, bill estimate, review tasks, and budget risk.
- Bill Center shows estimated card bills first, with real statement variance when available.
- Payment schedule details are behind row expansion or a detail drawer.
- Budget taxonomy uses two levels: budget group and budget item.

```ts
const numberFormatter = new Intl.NumberFormat("zh-TW", {
  maximumFractionDigits: 0
});

export function formatCurrency(value: number): string {
  return numberFormatter.format(value);
}

export function formatMonth(month: string): string {
  if (!/^\d{4}-(0[1-9]|1[0-2])$/.test(month)) {
    return "月份未設定";
  }

  return month;
}

export function formatPercent(value: number): string {
  return `${(value * 100).toFixed(1)}%`;
}

export function formatVariance(value: number): string {
  if (value > 0) {
    return `+${formatCurrency(value)}`;
  }

  return formatCurrency(value);
}
```

Create `apps/web/src/lib/bill-calculations.ts`:

```ts
export type BillComparisonInput = {
  estimatedAmount: number;
  statementAmount?: number;
};

export function getDisplayedBillAmount(input: BillComparisonInput): number {
  return input.statementAmount ?? input.estimatedAmount;
}

export function getStatementVariance(input: BillComparisonInput): number | null {
  if (input.statementAmount === undefined) {
    return null;
  }

  return input.statementAmount - input.estimatedAmount;
}
```

```tsx
                <td>{bill.statementAmount ? formatCurrency(bill.statementAmount) : "未輸入"}</td>
```
