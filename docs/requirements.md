# 需求文件

本文件描述現行 Accounting Automation MVP 的產品需求。完整交接入口請先閱讀 [產品開發交接文件](product-development-guide.md)。

## 產品目標

建立一套個人記帳系統，讓每筆消費只需記錄一次，但能同時滿足兩種管理視角：

- 預算控管：依消費日立即扣年度與月份預算。
- 現金流管理：依實際付款日或信用卡付款日反映現金流。

系統目前採用 Google Apps Script Web App + Google Sheet 資料庫，GitHub 只保存程式碼、文件、模板與匿名範例，不保存真實帳務資料。

## 核心原則

消費日扣預算，付款日扣現金流。

例：2 月刷卡購買 12,000 元日用品並分 3 期付款時，2 月日用品預算立即動支 12,000 元；現金流則依信用卡付款日拆到後續 3 個月份。

## 使用者需求

### 發票消費

- 使用者可從財政部下載發票 CSV，或從 Excel/CSV 複製發票明細貼上。
- 系統匯入後不得直接入帳，必須先進入待確認清單。
- 待確認清單需顯示消費日、店家、品項、金額、預算項目、支付方式、信用卡。
- 使用者可批次確認入帳、批次刪除、批次確認並寫入店家支付規則。
- 重複匯入同一張發票同一明細時，系統需自動略過。
- 發票內的折扣、點數折抵、零元明細也要保留，允許金額為 0 或負數。

### 無發票與海外消費

- 使用者可單筆輸入無發票消費。
- 使用者可批次匯入手動消費 CSV 或貼上來源值，支援購物車多品項。
- 手動批次匯入可包含標題列，也可省略標題列並依預設欄位順序解析。
- 手動批次匯入若只提供年月，例如 `2026/01`，系統以該月 1 日作為消費日，預算月份仍為該月。
- 手動批次匯入可包含折扣或調整項，因此允許 0 或負數金額。
- 單筆手動輸入仍必須大於 0。

### 預算分類

- 預算項目必須來自 `BudgetItems.budget_item`，不得任意建立自由文字分類。
- `BudgetItems.is_valid_expense_item` 為 true 的項目才可作為消費分類。
- 發票匯入後預設分類順序為：來源註記 > 店家支付規則 > 店家 + 品項規則 > `24. 餐費 + 現金`。
- 所有匯入發票仍標記為 `needs_review`，由使用者確認。

### 支付與分期

- 支付方式分為 `cash` 與 `credit_card`。
- 信用卡名稱目前支援玉山、聯邦、國泰、富邦、中信。
- 分期付款先支援平均分期，最後一期吸收除不盡的尾差。
- 付款排程可於對帳時手動修正金額與狀態。

### 收入與現金流

- 收入記錄在 `IncomeSchedule`。
- 薪資排程可建立每月預估收入。
- 實際入帳後可在 UI 將收入標記為 `received`，金額不同時可改成 `corrected` 並修正金額。
- 現金流總覽公式為：月收入 - 現金支出 - 信用卡付款。
- 第一版不處理期初現金餘額、銀行帳戶餘額、帳戶間轉帳。

## 目前功能範圍

已實作：

- Google Sheet 11 張資料表初始化。
- 預算總覽與單筆消費後預算影響。
- 現金流總覽、每月收入預估與每月帳單預估。
- 手動單筆消費。
- 手動批次匯入。
- 財政部發票匯入與待確認清單。
- 待確認發票分頁載入與「載入更多」。
- 待確認發票批次確認、批次刪除、批次匯入店家支付規則。
- 店家支付規則與店家 + 品項規則；店家支付規則含 `merchant_display_name` 供人工辨識。
- 發票重複檢查。
- 付款排程明細的對帳/修正。
- 收入排程的入帳/修正。

未完成或後續加強：

- 更完整的信用卡帳單對帳報表與取消消費沖銷流程。
- 自動規則學習：最近 5 次有 4 次一致才提升為預設規則。
- 期初餘額、銀行帳戶、轉帳規劃。
- GitHub `rules/` 範例 CSV 與 11 張正式表完全一致化。
